# v2.0.0 发布说明

## 概要
- 本版本为核心功能重大升级，增强了认证兼容性、模型访问控制与本地化服务能力。
- 引入 OpenSpec 规范框架，建立标准化的需求管理与开发流程。

## 关键更新
1. **认证兼容增强**：新增 Azure-style 多种认证方式支持（Bearer、api-key 头部/查询参数），客户端无需额外代理专用头部，转发时自动替换为服务器凭证。
2. **本地模型列表**：代理直接从配置返回模型列表（`/openai/deployments`、`/openai/models`、`/models`、`/v1/models`），不再转发上游，支持权限过滤和去重。
3. **模型访问控制**：每个 target 支持配置 `allowed_models` 白名单，拒绝未授权模型请求；基于模型名称自动选择目标，支持负载均衡。
4. **API 版本强制**：每个 target 配置 `default_api_version`，强制覆盖客户端提供的 `api-version`，确保请求使用指定版本。
5. **OpenSpec 规范框架**：新增需求文档标准化结构，为后续功能迭代提供统一的规范依据和测试指南。
6. **配置验证与日志**：强制 target 配置 Azure 凭证，优化错误提示（如 `context deadline exceeded`），完善配置管理。

## 升级与部署建议
- 配置文件需为每个 target 添加 `allowed_models` 和 `default_api_version` 字段。
- 客户端可使用 Azure 标准认证方式（Bearer 或 api-key），新增端点无需代码改动。
- 运行 `make build` 或使用 `docker-compose up -d` 部署，验证模型列表和认证功能。
- 推送标签 `v2.0.0` 至远端后，可在仓库 Release 页面发布正式说明。

## 破坏性变更
- 无（所有改动向后兼容）
