# 部署要求 — 192.168.33.110

本文档针对 `192.168.33.110`（主机名 `pc01260`）上 LLMs Proxy 的实际部署做汇总说明。

## 1. 主机环境

| 项目 | 值 |
|---|---|
| 操作系统 | Ubuntu 24.04.1 LTS (Noble Numbat), x86_64 |
| CPU 核心 | 24 |
| 内存 | 32 GB |
| 磁盘 | 1.8 TB（LVM，`/DATA` 与 `/` 同一逻辑卷） |
| Docker | 29.1.2（containerd image store） |
| Docker Compose | v5.0.0 |

## 2. 部署路径

```
/DATA/AppData/llms_proxy/          # 项目根目录（含完整源码）
├── docker-compose.yml             # Compose 定义
├── config/                        # 配置目录 → 容器内 /etc/llms-proxy
│   └── config.json                # 运行时配置
├── data/                          # 数据目录 → 容器内 /var/lib/llms-proxy
│   └── llms-proxy.db              # bbolt 数据库（客户端、用量、审计等）
├── logs/                          # 日志目录 → 容器内 /var/log/llms-proxy
│   ├── access.log
│   └── error.log
└── deploy/docker/Dockerfile       # 构建用 Dockerfile
```

## 3. Docker Compose 配置

```yaml
services:
  llms-proxy:
    container_name: llms-proxy
    build:
      context: .
      dockerfile: deploy/docker/Dockerfile
    image: llms-proxy:latest
    restart: unless-stopped
    environment:
      - SERVER_BIND=0.0.0.0:8000
      - LOG_LEVEL=info
    ports:
      - 8000:8000
    volumes:
      - ./config:/etc/llms-proxy        # 可读写，admin 保存配置需要写入权限
      - ./data:/var/lib/llms-proxy
      - ./logs:/var/log/llms-proxy
```

### 关键说明

- **config 目录不能挂载为 `:ro`**。容器进程以 `llmsproxy`（uid=100）用户运行，admin 界面保存目标/配置时需要写入 `config.json`。如果挂载为只读，运行时修改只在内存生效，重启后丢失，并报错：
  ```
  config applied at runtime but failed to persist to disk; changes will be lost on restart
  ```
- 服务直接通过局域网 IP `192.168.33.110:8000` 访问，**不经过 nginx 反向代理**。

## 4. 文件权限要求

容器进程以非 root 用户运行：

```
uid=100(llmsproxy) gid=101(llmsproxy)
```

宿主机上对应的挂载目录必须确保该 uid 有读写权限：

```bash
# config 目录：属主设为容器用户，保证 admin 可以持久化配置
chown -R 100:101 /DATA/AppData/llms_proxy/config/

# data 和 logs 目录：需要读写权限（默认 777 已满足）
# 如果权限不够，执行：
chown -R 100:101 /DATA/AppData/llms_proxy/data/
chown -R 100:101 /DATA/AppData/llms_proxy/logs/
```

> **注意**：uid 100 在宿主机上可能显示为 `dhcpcd` 等无关用户名，这是正常的，不影响功能。

## 5. 网络配置

| 项目 | 值 |
|---|---|
| 监听地址 | `0.0.0.0:8000`（容器内） |
| 端口映射 | `8000:8000`（宿主机 → 容器） |
| 访问方式 | `http://192.168.33.110:8000` |
| 网络模式 | 默认 bridge（`llms_proxy_default`） |

**禁止**将容器接入 `nginx_default` 等外部网络，否则端口映射会失效导致服务不可达。

## 7. 常用运维操作

### 构建镜像

```bash
cd /DATA/AppData/llms_proxy
docker compose build
```

### 启动 / 重启

```bash
docker compose up -d          # 首次启动或重建
docker compose restart        # 仅重启（不重建容器）
```

### 查看日志

```bash
docker compose logs -f --tail 100
```

### 健康检查

```bash
curl http://192.168.33.110:8000/healthz
# 返回 {"status":"ok"} 表示正常
```

### 热加载配置

```bash
curl -X POST http://192.168.33.110:8000/admin/config/reload
```

### 数据备份

```bash
# 停止容器后复制 bbolt 数据库文件
docker compose stop
cp /DATA/AppData/llms_proxy/data/llms-proxy.db /备份路径/llms-proxy-$(date +%Y%m%d).db
docker compose start
```

## 8. 故障排查清单

| 现象 | 可能原因 | 排查命令 |
|---|---|---|
| 外部无法访问 8000 端口 | 容器端口映射丢失（被接入了错误的网络） | `docker inspect llms-proxy --format '{{json .HostConfig.PortBindings}}'`，应返回含 `8000` 的映射 |
| admin 保存配置报错 "failed to persist to disk" | config 目录无写入权限或挂载为 `:ro` | `docker exec llms-proxy touch /etc/llms-proxy/test && echo OK`，失败则修复权限 |
| 容器启动后立即退出 | 配置文件格式错误 | `docker compose logs` 查看具体错误 |
| 容器名冲突无法创建 | 旧容器残留 | `docker rm -f llms-proxy && docker compose up -d` |

## 9. 注意事项

1. **该机器的 Docker 使用 containerd image store**（`io.containerd.snapshotter.v1`），`docker save` 默认输出 OCI 格式。如需导出镜像给 QNAP 等设备，需使用 `skopeo` 转换为 Docker 传统格式，详见 `AGENTS.md` 中的说明。
2. 该主机上同时运行 nginx 反向代理容器（服务 chat/dify/termix 等域名），llms-proxy **不需要**通过该 nginx 转发，保持独立的端口映射即可。
3. `admin_session.secret` 当前为开发默认值，生产环境应更换为强随机字符串（至少 32 字符）。
4. 镜像体积约 34 MB（Alpine + 静态编译 Go 二进制），资源消耗极低。
